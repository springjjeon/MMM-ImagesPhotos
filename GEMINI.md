## 효과 규칙
- 전환 순서: 검정(black) → 페이드인 → 표시 → 페이드아웃.
- 기본 타이밍(권장): 검정 1000ms, 페이드인 1000ms, 표시 5000ms, 페이드아웃 1000ms.
- 효과 목록(권장): zoom, pan(up/down/left/right), zoom-pan, fade, grayscale, sepia, rotate — 각 효과는 성능을 고려해 GPU에서 처리하도록 `transform`/`opacity` 사용.
- **이동(Pan) 효과 규칙**: 이동 애니메이션 적용 시 이미지 가장자리가 보이지 않도록, `transform: scale(1.15)`와 같이 이미지를 미리 확대해야 함. 이동 거리는 확대된 여백 이내여야 함.
- 애니메이션 지속시간은 이미지 표시 시간과 충돌하지 않도록 설정.
- 배경색은 이미지의 평균 색을 사용하되, cross-origin 이미지의 경우 캔버스 읽기 실패가 발생할 수 있음.
  - 필요 시 이미지에 `crossOrigin = 'Anonymous'`를 설정하고 서버에 CORS 헤더(`Access-Control-Allow-Origin: *`)가 있는지 확인.

# 업데이트 기록

## [2026-01-01] EXIF 정보 표시 기능 추가

이 업데이트는 `MMM-ImagesPhotos` 모듈에 EXIF 메타데이터를 표시하는 기능을 추가하고 관련 UI/UX를 개선합니다.

### 1. EXIF 메타데이터 표시

사진의 EXIF 정보를 읽어와 화면에 표시하는 기능이 추가되었습니다. 표시되는 정보는 다음과 같습니다.

-   위치 (GPS)
-   촬영일
-   카메라 모델
-   상세 촬영 정보 (초점거리, 조리개, 셔터속도, ISO)

**주요 특징**
-   **동적 표시**: 각 정보는 해당 데이터가 사진의 EXIF에 존재할 경우에만 동적으로 표시됩니다. 모든 정보가 없는 경우, 메타정보 창은 나타나지 않습니다.
-   **UI 개선**: 텍스트 레이블을 이모지(📍, 🗓️, 📷, ⚙️)로 대체하여 깔끔한 인터페이스를 제공합니다. 정보 창은 화면 우측 하단에 위치합니다.

### 2. 위치 정보 표시 (리버스 지오코딩)

-   사진에 GPS 좌표가 포함된 경우, OpenStreetMap(Nominatim)의 무료 리버스 지오코딩 서비스를 사용하여 좌표를 실제 지역 이름으로 변환 후 표시합니다.
-   **요구사항**: 이 기능은 `node_helper.js`에서 외부 API를 호출하므로, MagicMirror 서버의 인터넷 연결이 필요합니다.
-   **언어**: API 호출 시 한국어(`ko`)를 우선 요청하여 지역 이름이 한국어로 표시될 수 있도록 시도합니다.

### 3. 날짜 및 시간 표시 형식

-   EXIF의 `DateTimeOriginal` 타임스탬프 값을 서버의 시간대(Timezone) 설정에 영향을 받지 않는 UTC 기준으로 변환합니다.
-   `YYYY년 MM월 DD일 HH시 MM분` 형식의 한국식 날짜/시간으로 표시하여 가독성을 높였습니다.

### 4. 상세 촬영 정보 표시

-   사진 전문가를 위한 상세 촬영 정보를 한 줄로 요약하여 제공합니다.
-   **표시 항목**:
    -   초점거리 (예: `50mm`)
    -   조리개 값 (예: `f/1.8`)
    -   셔터속도 (예: `1/125s`)
    -   ISO 값 (예: `ISO 400`)

### 5. `showExif` 설정 옵션 추가

사용자가 `config.js`에서 EXIF 정보 표시 여부를 선택할 수 있도록 `showExif` 옵션이 추가되었습니다.

- **기본값**: `true`
- **동작**:
  - `showExif: false`로 설정 시, EXIF 관련 모든 UI가 표시되지 않습니다.
  - 또한, `node_helper.js`에서 EXIF 파싱 및 리버스 지오코딩 API 호출 등 불필요한 백엔드 작업을 건너뛰어 성능을 향상시킵니다.
